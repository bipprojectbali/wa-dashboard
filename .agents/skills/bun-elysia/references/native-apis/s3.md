# Bun.s3 - Native S3 Client

Bun provides native bindings for S3-compatible object storage with a familiar web-standards API (similar to Blob/Response). Works with AWS S3, Cloudflare R2, Google Cloud Storage, MinIO, DigitalOcean Spaces, and more.

## Connection Setup

### Default Client (Environment Variables)

```typescript
import { s3 } from "bun";

// Automatically uses environment variables for credentials
const file = s3.file("path/to/file.txt");
```

**Environment Variables (in order of preference):**

| Variable | Fallback | Description |
|----------|----------|-------------|
| `S3_ACCESS_KEY_ID` | `AWS_ACCESS_KEY_ID` | Access key |
| `S3_SECRET_ACCESS_KEY` | `AWS_SECRET_ACCESS_KEY` | Secret key |
| `S3_REGION` | `AWS_REGION` | Region |
| `S3_ENDPOINT` | `AWS_ENDPOINT` | Custom endpoint |
| `S3_BUCKET` | `AWS_BUCKET` | Default bucket |
| `S3_SESSION_TOKEN` | `AWS_SESSION_TOKEN` | Session token (STS) |

### Custom S3Client

```typescript
import { S3Client } from "bun";

const client = new S3Client({
  accessKeyId: "your-access-key",
  secretAccessKey: "your-secret-key",
  bucket: "my-bucket",
  region: "us-east-1",
  endpoint: "https://s3.us-east-1.amazonaws.com"
  // sessionToken: "..." // Optional for temporary credentials
});

const file = client.file("path/to/file.txt");
```

## S3File Lazy References

`s3.file()` returns a lazy reference - no network request is made until you call a method that requires data.

```typescript
import { s3 } from "bun";

// Synchronous - zero network requests
const file = s3.file("data/config.json");

// Network request only happens when needed
const data = await file.json();  // Now fetches from S3
```

This design allows efficient handling of S3 objects without unnecessary requests.

## Reading Files

### Multiple Formats

```typescript
const file = s3.file("data.json");

// Read as JSON (parsed automatically)
const data = await file.json();

// Read as text
const text = await file.text();

// Read as bytes (Uint8Array)
const bytes = await file.bytes();

// Read as ArrayBuffer
const buffer = await file.arrayBuffer();
```

### Partial Reads (Range Requests)

```typescript
// Read only first 1KB
const header = await file.slice(0, 1024).text();

// Read specific byte range
const chunk = await file.slice(1000, 2000).bytes();

// Efficient for large files - only downloads requested bytes
```

### Streaming Large Files

```typescript
const file = s3.file("large-video.mp4");

// Get readable stream
const stream = file.stream();

// Process chunks as they arrive
for await (const chunk of stream) {
  console.log("Received chunk:", chunk.length);
}

// Pipe directly to HTTP response
return new Response(file.stream(), {
  headers: { "Content-Type": "video/mp4" }
});
```

## Writing Files

### Basic Writing

```typescript
import { s3, write } from "bun";

const file = s3.file("output.txt");

// Write string
await file.write("Hello, World!");

// Write with content type
await file.write(JSON.stringify(data), {
  type: "application/json"
});

// Write buffer/bytes
await file.write(new Uint8Array([1, 2, 3]));

// Write from Response
await file.write(new Response("content"));

// Write from Blob
await file.write(new Blob(["data"], { type: "text/plain" }));

// Alternative: use Bun.write
await write(s3.file("data.json"), JSON.stringify({ name: "John" }));
```

### Streaming Writes (Large Files)

For large files, use the writer for multipart uploads.

```typescript
const file = s3.file("large-file.bin");

const writer = file.writer({
  type: "application/octet-stream"
});

// Write chunks incrementally
writer.write(chunk1);
writer.write(chunk2);
writer.write(chunk3);

// Finalize the upload
await writer.end();
```

### Multipart Upload Configuration

```typescript
const writer = file.writer({
  type: "application/octet-stream",
  partSize: 10 * 1024 * 1024,  // 10MB parts (min 5MB)
  queueSize: 4                  // Concurrent upload parts
});

// Larger queueSize = faster uploads but more memory usage
```

## Presigned URLs

Generate time-limited URLs for secure access without exposing credentials.

### For Downloads (GET)

```typescript
const file = s3.file("private/document.pdf");

// Default: GET method, 24 hours expiration
const downloadUrl = file.presign();

// Custom expiration
const url = file.presign({
  expiresIn: 3600  // 1 hour in seconds
});
```

### For Uploads (PUT)

```typescript
// Generate URL for client-side uploads
const uploadUrl = s3.file("uploads/user-avatar.jpg").presign({
  expiresIn: 600,        // 10 minutes
  method: "PUT",
  type: "image/jpeg",
  acl: "public-read"     // Make uploaded file public
});

// Client can upload directly to S3:
// await fetch(uploadUrl, { method: "PUT", body: imageBlob });
```

### ACL Options

| ACL | Description |
|-----|-------------|
| `private` | Owner only (default) |
| `public-read` | Anyone can read |
| `public-read-write` | Anyone can read/write |
| `authenticated-read` | AWS authenticated users |
| `bucket-owner-read` | Bucket owner can read |
| `bucket-owner-full-control` | Bucket owner gets full control |

### Presign Options

```typescript
const url = file.presign({
  expiresIn: 86400,    // Seconds until expiration (default: 24 hours)
  method: "GET",       // HTTP method: GET, PUT, DELETE, HEAD, POST
  type: "image/png",   // Content-Type for uploads
  acl: "private"       // Access control list
});
```

## File Management

### Check Existence

```typescript
const exists = await file.exists(); // boolean
```

### Get Metadata

```typescript
const stat = await file.stat();
// {
//   etag: "\"abc123def456\"",
//   lastModified: Date,
//   size: 1234,
//   type: "application/json"
// }
```

### Get Size

```typescript
const size = await file.size(); // bytes
```

### Delete Files

```typescript
await file.delete();

// Or using unlink
await file.unlink();
```

### List Objects

```typescript
// List all objects in bucket
const files = await s3.list();

// With prefix (like folder filtering)
const uploads = await s3.list({
  prefix: "uploads/",
  maxKeys: 100
});

// Pagination for large buckets
let continuationToken;
do {
  const result = await s3.list({
    prefix: "logs/",
    maxKeys: 1000,
    continuationToken
  });

  for (const file of result.contents) {
    console.log(file.key, file.size, file.lastModified);
  }

  continuationToken = result.nextContinuationToken;
} while (continuationToken);
```

## Static Methods

For one-off operations without creating a client instance.

```typescript
import { S3Client } from "bun";

const credentials = {
  accessKeyId: "...",
  secretAccessKey: "...",
  bucket: "my-bucket",
  endpoint: "..."
};

// Write directly
await S3Client.write("path/file.txt", "content", credentials);

// Get file reference
const file = S3Client.file("path/file.txt", credentials);

// Presign URL
const url = S3Client.presign("path/file.txt", {
  ...credentials,
  expiresIn: 3600
});

// Check existence
const exists = await S3Client.exists("path/file.txt", credentials);

// Get size
const size = await S3Client.size("path/file.txt", credentials);

// Delete
await S3Client.delete("path/file.txt", credentials);

// List objects
const files = await S3Client.list({ prefix: "uploads/" }, credentials);
```

## Best Practices

### 1. Use Environment Variables

```typescript
// Good - credentials from environment
import { s3 } from "bun";
const file = s3.file("config.json");

// Avoid hardcoding credentials
```

### 2. Stream Large Files

```typescript
// Good - streams without loading entire file in memory
return new Response(s3.file("large-video.mp4").stream());

// Avoid for large files - loads entire file in memory
const content = await s3.file("large-video.mp4").arrayBuffer();
```

### 3. Use Presigned URLs for Client Uploads

```typescript
// Good - client uploads directly to S3
app.get("/upload-url", () => {
  const url = s3.file(`uploads/${crypto.randomUUID()}.jpg`).presign({
    method: "PUT",
    expiresIn: 300,
    type: "image/jpeg"
  });
  return Response.json({ uploadUrl: url });
});

// Avoid - proxying large files through your server
app.post("/upload", async (req) => {
  const file = await req.blob();
  await s3.file("uploads/file.jpg").write(file); // Uses server memory/bandwidth
});
```

### 4. Use Partial Reads When Possible

```typescript
// Good - only download what you need
const header = await s3.file("large.csv").slice(0, 1000).text();

// Avoid - downloading entire file for metadata
const full = await s3.file("large.csv").text();
const header = full.slice(0, 1000);
```

### 5. Handle Errors Appropriately

```typescript
try {
  const data = await s3.file("config.json").json();
} catch (error) {
  if (error.code === "NoSuchKey") {
    // File doesn't exist - handle gracefully
    return defaultConfig;
  } else if (error.code === "AccessDenied") {
    console.error("Permission denied to S3 bucket");
  } else if (error.code === "ERR_S3_MISSING_CREDENTIALS") {
    console.error("S3 credentials not configured");
  }
  throw error;
}
```

### 6. Lazy File References for Conditional Access

```typescript
// Good - no request if condition is false
const file = s3.file("optional-config.json");
if (needsConfig) {
  const config = await file.json();
}

// The file reference costs nothing until accessed
```

## Error Codes

| Code | Description |
|------|-------------|
| `ERR_S3_MISSING_CREDENTIALS` | No credentials provided |
| `ERR_S3_INVALID_SIGNATURE` | Signature mismatch |
| `ERR_S3_INVALID_ENDPOINT` | Invalid endpoint URL |
| `NoSuchKey` | Object not found |
| `NoSuchBucket` | Bucket not found |
| `AccessDenied` | Permission denied |
| `InvalidAccessKeyId` | Invalid access key |
| `SignatureDoesNotMatch` | Secret key mismatch |

## S3 Protocol URLs

Bun supports `s3://` protocol URLs directly.

```typescript
// Fetch with S3 URL
const response = await fetch("s3://my-bucket/file.txt", {
  s3: {
    accessKeyId: "...",
    secretAccessKey: "..."
  }
});
const text = await response.text();

// Bun.file with S3 URL
const file = Bun.file("s3://my-bucket/config.json");
const config = await file.json();
```
